# Database Restore Process

This guide explains how to restore a tenant's PostgreSQL database from the automated backups.
The automated backups are stored on a dedicated PersistentVolumeClaim (PVC) named `<tenant>-backup-storage` and are generated by the daily CronJob.

## 1. Identify Available Backups

Since the backup PVC is mounted to the backup CronJob, we can spin up a temporary pod to view the contents of the backup directory.

```bash
TENANT_NAME="ahmad" # Replace with actual tenant name

# Create a temporary pod to access the backup volume
cat <<EOF | kubectl apply -f -
apiVersion: v1
kind: Pod
metadata:
  name: ${TENANT_NAME}-backup-access
  namespace: ${TENANT_NAME}
spec:
  containers:
  - name: utils
    image: postgres:14-alpine
    command: ["sleep", "3600"]
    volumeMounts:
    - name: backup-storage
      mountPath: /backup
  volumes:
  - name: backup-storage
    persistentVolumeClaim:
      claimName: ${TENANT_NAME}-backup-storage
EOF

# Wait for the pod to become ready
kubectl wait --for=condition=Ready pod/${TENANT_NAME}-backup-access -n ${TENANT_NAME}

# List available backup files
kubectl exec -it ${TENANT_NAME}-backup-access -n ${TENANT_NAME} -- ls -lh /backup
```

You should see files named like `ahmad_backup_20260521_020000.sql`. Choose the file you want to restore.

## 2. Restore the Database

Ensure the target database is ready to accept connections. We will use `psql` to restore the `.sql` dump.

```bash
# Set the desired backup file
BACKUP_FILE="ahmad_backup_20260521_020000.sql"

# Get the DB password
DB_PASSWORD=$(kubectl get secret ${TENANT_NAME}-db-credentials -n ${TENANT_NAME} -o jsonpath="{.data.POSTGRES_PASSWORD}" | base64 --decode)

# Get the DB name
DB_NAME=$(kubectl get configmap ${TENANT_NAME}-config -n ${TENANT_NAME} -o jsonpath="{.data.DATABASE_NAME}")

# Run the restore process using the temporary access pod
# Note: The database service is named <tenant>-db-svc
kubectl exec -it ${TENANT_NAME}-backup-access -n ${TENANT_NAME} -- sh -c "PGPASSWORD=${DB_PASSWORD} psql -h ${TENANT_NAME}-db-svc -U postgres -d ${DB_NAME} < /backup/${BACKUP_FILE}"
```

## 3. Clean Up

Once the restore is verified and successful, delete the temporary access pod.

```bash
kubectl delete pod ${TENANT_NAME}-backup-access -n ${TENANT_NAME}
```

## Troubleshooting

- **Connection Refused**: Ensure the ERP database pod is running. Check with `kubectl get pods -n ${TENANT_NAME}`.
- **Permission Denied**: The backups are generated by the `postgres` user (UID 999). Make sure the restore command runs seamlessly using the provided `postgres:14-alpine` image.
